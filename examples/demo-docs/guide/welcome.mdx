This page lives in `examples/demo-docs/` inside the Portico repository. The renderer reads
`docs.json` and `*.mdx` from whatever directory `CONTENT_ROOT` points to.

## Resolution order

1. `CONTENT_ROOT` environment variable (if set)
2. `.content-root` pointer (after `CONTENT_GIT_REPO` clone in CI/Vercel)
3. Sibling `agentruntime-docs` checkout (AgentRuntime dev layout)
4. **`examples/demo-docs`** (this folder) — default for OSS clone-and-run

<Note>
  Production sites rebuild when content changes. Hosting does not read GitHub on every page view —
  it runs `npm run build` with your docs on disk, then serves the static output.
</Note>
